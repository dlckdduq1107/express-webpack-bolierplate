## branch 활용계획

    - J165 branch : upstream에서 fetch 받아오는 브랜치( 매 작업 마다 새로운 브랜치를 생성한다.)

    - feature branch : fetch받아온 브랜치에서 갈라져 나와 origin에 push하고 최종적으로 PR을 날리는 브랜치, fetch 받은 이후에는 새로운 브랜치를 생성해줘야 한다.

## 작업 시작시 Git 세팅(매일)

    - PR이 정상적으로 merge 되었는지 확인
    - J165로 checkout한다.
    - git fetch upstream J165
    - git rebase upstream/J165
    - git checkout -b 새로운 브랜치명
    - PR보낸후 작업한 내용이 있으면(PR이 merge 되기전 commit만 하고 push안했을때)
    - cherry-pick으로 가져오고 싶은 commit만 가져오기
    - 작업시작 ...
    - 새로운 브랜치로 commit, push
    - 새로운 브랜치에서 PR날림

### 백엔드

- 화면에서 붙힌 번호를 기준으로 관련 기능 태그(괄호 이용)
- 일단 로그인 한다고 가정
- 숙소 데이터는 미리 입력

        A. DB구조
            숙소리스트 테이블
                1. 숙소고유 ID(String)
                2. 숙소이름(String)
                3. 숙소 위치(String?number?)
                4. 대표이미지(String- 주소)
                5. 옵션(String)
                6. 평점과 후기(List-string) - [[평점,후기], ...]
                7. 1박요금(INT)
            회원리스트 테이블
                1. 회원고유 ID(String)
                2. 회원 로그인 아이디(String)
                3. 회원 로그인 비밀번호(String)
                4. 예약리스트(List-string) - [숙소고유ID, ...]
                5. 위시리스트(List-string) - [숙소고유ID, ...]

        B. 검색바
            (3) 요금
                숙소테이블 - 가격별 숙소개수 그릴때 사용
            (5) 검색버튼
                체크인/아웃 데이터, 요금데이터, 인원데이터 JSON POST

        C. 검색결과 및 지도
            (7) 숙소 테이블 - 숙소 위치 데이터로 숙소 정보 출력
            (8) 숙소 테이블 - 리스트 출력시, 대표이미지, 위치, 숙소이름, 옵션, 평점과 후기, 1박요금 데이터 사용
            (8) 회원 테이블 - 예약누르면, 회원의 숙소리스트에 위 데이터 저장

        D. 로그인(구현시 세부사항 추가)
            회원 테이블 - 세션과 쿠키 사용한 로그인
